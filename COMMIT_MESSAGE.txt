## Changes Summary

### 1. WorkflowExecutionRequest - Improved Signal and CommandResult documentation
- Added clear documentation for mutually exclusive Signal and CommandResult properties
- Clarified that Signal is for SignalWait triggers
- Clarified that CommandResult is for deferred/dispatched CommandWait triggers
- Improved property ordering and comments for better clarity

### 2. WorkflowStateDto - Replaced CancelledTokens with CancellationHistory
- Replaced `HashSet<string> CancelledTokens` with `List<CancellationHistoryEntry> CancellationHistory`
- CancellationHistory provides full audit trail with timestamps and reasons
- Better semantics: cancellation is history/audit data, not workflow state

### 3. New Files Created
- `Workflows.Abstraction/DTOs/CancellationHistoryEntry.cs`: Complete cancellation event structure with Token, CancelledAt, and Reason
- `Workflows.Abstraction/DTOs/CancellationHistoryExtensions.cs`: Helper extension methods for backward compatibility
- `WORKFLOW_DTO_REFACTORING.md`: Complete documentation of the refactoring
